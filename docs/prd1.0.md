
请根据以下 PRD 开发 PDF 智能闪卡生成系统

## 1. 需求背景
用户通过上传 PDF 文档，利用 AI 技术自动分析文档中的关键知识点、概念或考点，并将其转化为“问题-答案”格式的闪卡，旨在提高复习与记忆效率。

## 2. 功能流程拆解
   1. 文件上传与预处理：接收 PDF，验证文件完整性，进行 OCR（如有必要）或文本提取。
   2. 内容切片与分析：对长文本进行分段，利用 LLM（大语言模型）识别核心知识点。
   3. 闪卡内容生成：基于 Prompt 工程将知识点转化为 Q&A 对。
   4. 结果导出与管理：支持在线预览、编辑闪卡，并导出至 Anki 或 CSV 格式。

## 3. 详细功能设计
## 3.1 模块一：PDF 解析引擎
* 输入：支持本地上传（Max 50MB）或 URL 导入。
* 处理逻辑：
* 普通 PDF：直接使用工具（如 PyMuPDF, PDFPlumber）提取文字流。
   * 扫描版 PDF：调用 OCR 接口（如 [阿里云文档智能](https://www.aliyun.com/product/ai/docmind) 或 [TextIn](https://adg.csdn.net/69525ad65b9f5f31781badf7.html)）进行文字识别。
   * 结构识别：保留目录、标题层级，以便按章节生成闪卡。

## 3.2 模块二：AI 闪卡生成（核心）
* 核心逻辑：采用 RAG（检索增强生成）或长文本直接推理。
* Prompt 策略：
* 角色设定：你是一位资深教师，擅长从教材中提取考点。
   * 任务要求：请从以下文本中提取 5-10 个关键知识点，并改写为 Anki 格式的闪卡（正面：问题/概念，背面：简洁的解释）。
   * 输出约束：避免重复，确保答案准确且易于背诵。
* 智能筛选：用户可选择生成模式（如：定义解释、是非判断、案例分析）。

## 3.3 模块三：前端交互与管理
* 预览模式：双面卡片翻转效果。
* 编辑功能：用户可手动修改 AI 生成的错误内容或删除不重要的卡片。
* 分类标记：支持按 PDF 文件名自动打标签，或手动分类（如：医学、法考）。

## 3.4 模块四：导出与同步
* 格式支持：
* .apkg：直接导入 Anki 软件。
* .csv/.txt：通用格式。
* .pdf：可打印的卡片排版。

## 4. 技术架构建议
* 使用Next.js（App Router）实现无刷新页面切换。
* 存储：对象存储（OSS）存 PDF，向量数据库（如 Pinecone）存文本索引以支持超长文档解析。

## 5. 异常处理
* 解析失败：针对加密 PDF 或图片过多无法 OCR 的情况，给出明确提示。
* 生成超时：AI 处理长文档较慢，需支持异步任务处理及 WebSocket 进度条回传。
